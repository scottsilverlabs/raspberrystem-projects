LED MATRIX

In this project, we're going to introduce a new component that can be used for a variety of purposes, including building games.  It's called an LED Matrix, and like the name implies, it is basically a small device that contains lots of little LEDs (64 of them, to be exact).  This is what an LED Matrix looks like:

XXX

Included with the RaspberrySTEM Kit is a bag labeled, "LED Matrix RaspberrySTEM Cell."  This bag contains all the parts that you'll need to build and program with an LED Matrix.  In that bag you should find the following:

  • LED Matrix:  This actually consists of two parts that we've assembled for you -- the circuit board on the bottom and the LED display on the top.  You can pull the display from the circuit board, but do so carefully so that none of pins bend or break.

  • LED Matrix Cable:  This cable consists of six wires, with connectors on each end.  This is the cable that will be used to connect the LED Matrix to thebreadboard.

  • Binding Posts:  These two small posts and screws will allow you to lay the LED Matrix flat on the breadboard, as opposed to the default orientation of standing up.

For the next several projects, we'll be introducing nbew components that will require at least one-third of the breadboard space.  If you have enough room remaining on the breadboard, you can use that empty space; otherwise, we recommend removing some or all of the circuits you currently have on the breadboard to allow enough space for the next set of projects.

Note:  The wire attached to the RST pin on the LED Matrix will not get attached to anything at this point.  RST stands for "Reset," and this wire can be used to reset the LED Matrix (essentially powering it off and on) by briefly attaching the pin to 3.3V.

Testing the LED Matrix


from rstem.led_matrix import FrameBuffer

FrameBuffer()

    • A framebuffer is where the next image to be displayed on the screen is stored.  General usage:

      • erase()

      • draw stuff

      • show()

      • ... repeat

    • New code - a dot on the screen.

from rstem.led_matrix import FrameBuffer

fb = FrameBuffer()

x, y = (3, 3)

fb.erase()

fb.point(x, y)

fb.show()
