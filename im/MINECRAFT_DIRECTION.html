The Introduction

The Minecraft API on the Raspberry Pi does not have any direct way to determine which direction Steve is facing. This makes it essentially impossible to create code that allows you to modify the Minecraft world relative to what you are seeing in front of you on the screen. For example, without being able to programmatically tell what direction you're facing, you can't write a program that will do something directly in front of (or next to or behind) your character.

However, the RaspberrySTEM adds a couple new APIs that allows us to get the direction Steve is facing, and therefore allows us to implement additional features that you couldn't get with the Minecraft API by itself. Note that to use the functions below, we are doing some behind-the-scenes trickery with Minecraft, and because of that, you are likely to see the screen briefly go blank (up to a half second) when you call these functions -- unfortunately, that's the result of what we have to do to get these APIs to work.

get_direction()

The get_direction() function returns provides two angles - the first angle is your heading, similar to the angle provided by get_heading(). The second angle is the angle Steve is looking in the Y-axis (up or down). This function uses a different method to determine the angle, which is not as precise as get_heading().
