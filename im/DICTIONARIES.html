DICTIONARIES

In a previous concept, we discussed "Lists," how they work and what they are good for. If you recall, Lists are great for storing an ordered sequence of items. For example, a list of names or a list of values. In a List, the sequence of the data is important, and each piece of data is an independent entity.

Sometimes though, you want to be able to associate certain pieces of data with other pieces of data. For example, you may have a list of people, and you want to associate each person's age with each name.

Using a list to store this these associated pieces of data is very difficult and cumbersome. But, Python provides another mechanism that is perfect for storing data in this way -- it's called a Dictionary. Just like a regular Dictionary that is use to map words to their defintions, a Python Dictionary maps "keys" (the first column) to their "values" (the second column).
